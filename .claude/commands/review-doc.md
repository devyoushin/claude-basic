아래 기준으로 `$ARGUMENTS` 문서를 검토하고 개선점을 제안해 주세요.

## 검토 항목

### 1. 구조 완성도
- [ ] 필수 섹션(개요/원리/예시/토큰효율/주의사항) 모두 존재하는지

### 2. 예시 품질
- [ ] 실제 실행 가능한 프롬프트 예시인지
- [ ] 좋은 예시와 나쁜 예시를 비교했는지

### 3. 토큰 효율성
- [ ] 토큰 비용 관련 분석이 있는지
- [ ] `tokenops/` 시스템과 연계됐는지

### 4. 규칙 파일 연계
- [ ] `rules/` 디렉토리의 관련 규칙(T1-T5, P1-P5 등)이 참조됐는지

## 출력 형식

```
## 검토 결과: {파일명}

### 잘된 점
- ...

### 개선 필요 사항
1. [섹션명] 문제 → 개선 방안
```

검토 완료 후 `tokenops/log.md`를 업데이트해 주세요.
