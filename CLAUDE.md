## 응답 규칙

**형식:**
- 응답은 짧고 직접적으로. 불필요한 서론/마무리 없이.
- 코드는 항상 코드블록으로. 언어 명시 필수.
- 설명이 필요할 때만 설명. 요청하지 않은 설명 추가 금지.
- 마크다운 헤더는 최대 3단계(###)까지만.

**하지 말 것:**
- "좋은 질문입니다", "물론이죠" 같은 채우기 문구 금지.
- 요청하지 않은 리팩토링, 기능 추가, 개선 제안 금지.
- 이미 있는 내용을 다시 설명하는 반복 금지.
- 불확실한 토큰 수를 확정적으로 표현 금지. 항상 `~` 붙여서 추정치 표시.

## TokenOps 규칙 (필수 적용)

### 1. 세션 로그 기록 (모든 작업 후 의무)

작업이 완료될 때마다 `tokenops/log.md`에 아래 형식으로 한 줄 추가:

```
| YYYY-MM-DD | S{다음번호} | {작업유형} | {모델} | ~{input} | ~{cache} | ~{output} | ~{합계} | ~${비용} | {한 줄 메모} |
```

**작업유형 태그:** `DOC` `CODE` `DEBUG` `REVIEW` `RESEARCH` `CHAT`
단가 및 계산식 → `tokenops/budget.md` 참조

### 2. 대시보드 업데이트

log.md에 항목 추가 후 `tokenops/dashboard.md`의 아래 항목을 갱신:
- 총 세션 수, 총 토큰 합계, 총 추정 비용
- 해당 작업유형의 세션 수 및 비용
- TOP 5 고비용 세션 (필요 시)
- 월별 추이

### 3. 예산 초과 감지

작업 완료 후 당일 누적 비용이 `tokenops/budget.md`의 경고 임계치($0.80/일)를 넘으면
응답 마지막에 아래 경고를 출력:

```
⚠️ TokenOps 경고: 오늘 누적 비용이 ~$X.XX로 일일 경고 임계치($0.80)를 초과했습니다.
→ Haiku 모델 전환 또는 /clear 후 새 세션 시작을 검토하세요.
```

### 4. 고비용 작업 사전 경고

단일 작업의 예상 토큰이 5,000을 초과할 것으로 예상되면 실행 전에 알림:
```
⚠️ 이 작업은 고토큰 작업입니다. (예상 ~X,000 토큰)
진행하시겠습니까? (범위 축소 또는 Agent 위임을 권장합니다)
```

### 5. 새 규칙 추가 시

`rules/` 하위 해당 파일에 추가 → `rules/README.md` 요약표 업데이트
