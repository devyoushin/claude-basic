## 전체 규칙 요약

### 토큰 (T)
- **T1** 출력 형식 명시 (표/코드만/N줄)
- **T2** 새 주제는 `/clear` 후 새 대화
- **T3** 파일은 줄 번호로 범위 지정
- **T4** 조건을 첫 메시지에 완전하게
- **T5** 작업 난이도에 맞는 모델 선택

### 프롬프트 (P)
- **P1** 복잡한 작업엔 역할 부여
- **P2** 원하는 형식은 Few-shot으로 보여주기
- **P3** CoT("단계별로")는 복잡한 문제에만
- **P4** 부정문 대신 긍정문으로 지시
- **P5** 반복 컨텍스트는 CLAUDE.md에 저장

### Agent (A)
- **A1** 탐색은 Explore Agent에게 위임
- **A2** 독립 작업은 병렬 실행
- **A3** 위험한 작업은 Worktree 격리
- **A4** 긴 작업은 Background로
- **A5** Agent 결과는 핵심 부분 검증

---

### 캐싱 (C)
- **C1** 긴 시스템 프롬프트는 `cache_control`로 마킹
- **C2** 같은 문서를 반복 질문할 때 캐싱 활용
- **C3** 캐시할 내용은 메시지 앞쪽에 배치
- **C4** 캐시 유효 시간은 5분 (연속 작업에서 효과적)
- **C5** Claude Code는 CLAUDE.md를 자동 캐싱

### 워크플로우 (W)
- **W1** 버그 수정: 현상 → 범위 → 원인 → 수정 순서로
- **W2** 새 기능: 설계 먼저 승인 후 단계별 구현
- **W3** 코드 리뷰: 범위 + 관점 + 형식 지정
- **W4** 코드 이해: Explore Agent로 구조 파악 후 질문
- **W5** 반복 작업: 예시 1개 확정 후 일괄 처리

### 실수 (M)
- **M1** "더 잘 해줘" 루프 → 기준 명시로 해결
- **M2** 대화 중간 주제 전환 → `/clear` 후 새 대화
- **M3** Claude를 파일 시스템처럼 쓰기 → 결과물은 직접 저장
- **M4** "자세히", "모든" 남발 → 필요한 수준 명시
- **M5** 에러 메시지 없이 "안 된다" → 에러 전문 붙여넣기
- **M6** 승인 없이 계속 진행 → 단계별 확인
- **M7** 모든 것을 Claude에게 → 검색/문서/실행으로 해결될 것은 직접

## 규칙 추가 방법

1. 해당 카테고리 파일에 `RULE-XX: 제목` 형식으로 추가
2. 새 카테고리가 필요하면 파일 새로 생성 후 이 README에 등록
3. 규칙이 바뀌면 이 요약표도 함께 업데이트
