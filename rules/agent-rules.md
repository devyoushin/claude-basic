# Agent 사용 규칙

> 카테고리: Agent / Subagent
> 참고 가이드: ../claude-guide.md#5-agent-안에서-agent-사용하기-subagent

---

## RULE-A1: 탐색 작업은 Explore Agent에게 위임하라

**규칙:** 코드베이스를 파악하거나 파일을 검색할 때는 직접 하지 말고 Explore Agent를 써라.

**이유:** 대규모 검색 결과가 메인 컨텍스트에 쌓이지 않음 → 메인 대화 토큰 절약

**적합한 작업:**
- "이 함수가 어디서 호출되는지 찾아줘"
- "API 엔드포인트 목록 뽑아줘"
- "이 변수명이 프로젝트 전체에서 어떻게 쓰이는지"

---

## RULE-A2: 독립적인 작업은 병렬로 실행하라

**규칙:** 서로 의존하지 않는 작업은 여러 Agent에게 동시에 보내라.

```
✅ 병렬 가능:
  - Agent1: "프론트엔드 코드 분석"
  - Agent2: "백엔드 코드 분석"
  → 동시 실행, 시간 절반

❌ 병렬 불가 (순서 의존):
  1. Explore Agent로 코드 파악
  2. (결과를 받은 후) Plan Agent로 설계
  3. (설계 후) 코드 작성
```

---

## RULE-A3: 위험한 작업은 Worktree로 격리하라

**규칙:** 실험적이거나 되돌리기 어려운 작업은 `isolation: "worktree"` 옵션을 써라.

**이유:**
- 임시 git worktree에서 실행
- 변경 없으면 자동 삭제
- 메인 브랜치에 영향 없음

**적합한 작업:**
- 대규모 리팩토링 시도
- 새 라이브러리 마이그레이션 테스트
- 파괴적 변경(breaking change) 실험

---

## RULE-A4: 오래 걸리는 작업은 Background로 보내라

**규칙:** 결과를 즉시 필요로 하지 않는 작업은 백그라운드로 실행하고 다른 작업을 진행하라.

```
✅ 적합:
  - 전체 테스트 실행
  - 대용량 파일 분석
  - 빌드 검증

사용법: run_in_background: true
→ 완료되면 자동 알림, 그동안 다른 작업 가능
```

---

## RULE-A5: Agent 결과는 직접 확인 후 신뢰하라

**규칙:** Subagent의 결과물은 그대로 신뢰하기 전에 핵심 부분을 검증하라.

**이유:** Agent는 잘못된 경로나 존재하지 않는 함수를 보고할 수 있음

**체크리스트:**
- [ ] 파일 경로가 실제로 존재하는가?
- [ ] 함수명이 현재 코드에 있는가?
- [ ] 제안된 변경이 기존 코드와 충돌하지 않는가?
