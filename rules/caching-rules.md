# 프롬프트 캐싱 규칙

> 카테고리: 캐싱 (Prompt Caching)
> 관련 문서: ../claude-guide.md

---

## 개념: 프롬프트 캐싱이란?

같은 input(시스템 프롬프트, 긴 문서 등)을 반복해서 보낼 때
Claude API가 해당 부분을 **캐시**해두고 다음 요청에 재사용하는 기능.

```
일반 요청:  [시스템 프롬프트 1,000토큰] + [질문 50토큰] → 매번 1,050토큰 과금
캐시 요청:  [캐시 히트 1,000토큰] + [질문 50토큰] → 캐시 부분은 90% 할인
```

캐시된 토큰은 일반 input 토큰 대비 **약 90% 저렴**.

---

## RULE-C1: 긴 시스템 프롬프트는 캐시 마킹하라

**규칙:** 시스템 프롬프트가 길고 반복 사용된다면 `cache_control`을 써라.

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[
        {
            "type": "text",
            "text": "당신은 Python 전문가입니다. [... 긴 내용 ...]",
            "cache_control": {"type": "ephemeral"}  # ← 캐시 마킹
        }
    ],
    messages=[{"role": "user", "content": "질문"}]
)

# 캐시 사용 여부 확인
print(response.usage.cache_creation_input_tokens)  # 캐시 생성 토큰
print(response.usage.cache_read_input_tokens)       # 캐시 읽기 토큰 (저렴)
```

## RULE-C2: 긴 문서 분석은 캐싱 최우선 후보다

**규칙:** 같은 문서를 여러 번 질문할 때는 반드시 캐싱을 활용하라.

```
❌ 비효율:
  [문서 10,000토큰] + "1번 질문" → 10,050토큰
  [문서 10,000토큰] + "2번 질문" → 10,050토큰  (문서를 또 전송)
  [문서 10,000토큰] + "3번 질문" → 10,050토큰
  합계: 30,150토큰

✅ 캐싱:
  [문서 10,000토큰 캐시 생성] + "1번 질문" → 10,050토큰
  [캐시 히트 1,000토큰(할인)] + "2번 질문" → 1,050토큰
  [캐시 히트 1,000토큰(할인)] + "3번 질문" → 1,050토큰
  합계: ~12,150토큰 (약 60% 절약)
```

## RULE-C3: 캐시는 대화 초반부에 위치시켜라

**규칙:** 캐시할 내용은 항상 메시지의 앞쪽(시스템 프롬프트, 첫 메시지)에 배치하라.

**이유:** Claude API 캐싱은 prefix 기반 — 뒤쪽이 바뀌면 앞쪽 캐시는 유지되지만,
앞쪽이 바뀌면 뒤쪽 전체 캐시가 무효화됨.

```
[캐시O] 시스템 프롬프트
[캐시O] 참고 문서 (고정)
[캐시X] 사용자 질문 (매번 바뀜)  ← 항상 맨 뒤
```

---

## RULE-C4: 캐시 유효 시간을 인지하라

**규칙:** 캐시는 생성 후 **5분** 동안만 유지된다. 5분 이상 간격이면 재생성됨.

**실전 적용:**
- 짧은 세션(코딩 작업, 문서 분석)에서 효과 최대
- 하루에 한 번 쓰는 용도라면 캐시 효과 없음
- 동일 문서를 연속해서 여러 번 질문할 때 활용

---

## RULE-C5: Claude Code에서의 캐싱

**규칙:** Claude Code는 CLAUDE.md와 대화 히스토리를 자동으로 캐싱한다.

- CLAUDE.md가 길어져도 매 턴마다 full 토큰이 과금되지 않음
- 단, CLAUDE.md를 자주 수정하면 캐시가 무효화되므로 안정적으로 유지할 것
- `/cost` 명령으로 캐시 적중률 확인 가능
