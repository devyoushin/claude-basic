# 토큰 비교: 같은 목적, 다른 질문법

> 작성일: 2026-04-16
> 목적: 질문 방식에 따라 토큰이 얼마나 차이나는지 실제 예시로 비교

---

## 읽는 법

각 예시마다 "낭비형"과 "절약형"을 비교합니다.
토큰 수는 실측 기반 추정치입니다.

```
[낭비형] input + output = 총 토큰
[절약형] input + output = 총 토큰
절약량: XX% 감소
```

---

## 예시 1. 코드 설명 요청

### 낭비형 질문
```
안녕하세요! 저는 Python을 배우고 있는 초보 개발자인데요,
제가 아래 코드를 작성해봤는데 이 코드가 어떻게 동작하는지
자세하게 설명해주실 수 있으신가요? 각 줄마다 무슨 일이
일어나는지, 왜 이렇게 작성하는지, 그리고 혹시 더 좋은
방법이 있다면 그것도 알려주시고, 초보자도 이해할 수 있게
쉽게 설명해 주시면 감사하겠습니다!

def add(a, b):
    return a + b
```
**Input: ~105 토큰**

Claude 응답 (장황한 설명):
```
안녕하세요! Python 학습 중이시군요, 정말 잘 하고 계세요!
이 코드를 함께 살펴보겠습니다.

def add(a, b):
  → 이 줄은 "add"라는 이름의 함수를 정의합니다.
    Python에서 함수는 def 키워드로 시작하며...
    a와 b는 매개변수(parameter)라고 부르는데...

return a + b:
  → return은 함수가 값을 돌려줄 때 사용합니다...

더 좋은 방법으로는 타입 힌트를 추가할 수도 있어요:
def add(a: int, b: int) -> int:
    return a + b

초보자분들이 자주 헷갈리는 부분인데, 함수란...
```
**Output: ~280 토큰**
**총합: 385 토큰**

---

### 절약형 질문
```
아래 코드 한 줄 요약:

def add(a, b):
    return a + b
```
**Input: ~25 토큰**

Claude 응답:
```
두 수를 더해 반환하는 함수.
```
**Output: ~12 토큰**
**총합: 37 토큰**

---

> **절약량: 90% 감소 (385 → 37 토큰)**
> 핵심: 원하는 출력 형식("한 줄 요약")을 명시하면 Claude가 장황하게 답하지 않음

## 예시 2. 파일 전체 vs 필요한 부분만

### 낭비형: 파일 전체 읽기
```
user_service.py 파일 읽고 버그 찾아줘
```

파일 내용 (300줄, 약 2,400 토큰이 input에 추가됨):
```python
# 1~50줄: import 및 설정
# 51~200줄: 관계없는 다른 메서드들
# 201~250줄: 문제의 함수 (실제로 분석이 필요한 부분)
# 251~300줄: 유틸리티 함수들
```
**Input: ~2,500 토큰 (파일 전체 + 질문)**
**Output: ~200 토큰**
**총합: 2,700 토큰**

---

### 절약형: 범위 지정
```
user_service.py의 201~250줄만 읽고 버그 찾아줘
```
**Input: ~180 토큰 (해당 부분만)**
**Output: ~200 토큰**
**총합: 380 토큰**

---

> **절약량: 86% 감소 (2,700 → 380 토큰)**
> 핵심: 어디가 문제인지 알면, 전체를 Claude에게 넘기지 말 것

## 예시 3. 대화 히스토리 누적

### 낭비형: 한 대화에서 계속 질문

```
[Turn 1] "Python이 뭔가요?" → 500토큰 대화
[Turn 2] "리스트는요?"     → 1,000토큰 누적 (Turn1 히스토리 포함)
[Turn 3] "딕셔너리는요?"   → 1,800토큰 누적
[Turn 4] "함수는요?"       → 2,700토큰 누적
[Turn 5] "클래스는요?"     → 3,800토큰 누적
```

**Turn 5 시점 Input: 3,800 토큰** (이전 대화 전부 포함)
(5개 질문 전체 합산: 약 9,800 토큰 소모)

---

### 절약형: 주제별로 새 대화 시작

```
[새 대화 1] "Python 리스트 3줄 요약"  → 60토큰
[새 대화 2] "Python 딕셔너리 3줄 요약" → 60토큰
[새 대화 3] "Python 함수 3줄 요약"    → 60토큰
[새 대화 4] "Python 클래스 3줄 요약"  → 60토큰
```

**4개 질문 전체 합산: 약 240 토큰**

---

> **절약량: 97% 감소 (9,800 → 240 토큰)**
> 핵심: 대화가 길어질수록 이전 내용 전체가 매 턴마다 input에 포함됨
> Claude Code에서는 `/clear`로 히스토리 초기화

## 예시 4. 모호한 요청 vs 명확한 요청

### 낭비형: 모호해서 재시도 발생

```
[1차] "로그인 만들어줘"
→ Claude: 어떤 언어? 어떤 DB? JWT? 세션?
  (되묻기 응답 ~150토큰 낭비)

[2차] "아 FastAPI로요"
→ Claude: 사용자 테이블 구조는? 비밀번호 해싱은?
  (또 되묻기 ~120토큰 낭비)

[3차] "그냥 기본적인걸로요"
→ Claude: 드디어 코드 작성 (~600토큰)
```

**총합: ~1,800 토큰** (왕복 3회)

---

### 절약형: 조건을 한 번에 명시

```
FastAPI 로그인 엔드포인트 작성.
- POST /login
- 입력: username, password (JSON body)
- DB: SQLite + SQLAlchemy
- 비밀번호: bcrypt 해싱
- 성공 시 JWT 반환, 실패 시 401
- 코드만, 설명 없이
```

**Input: ~80 토큰**
**Output: ~450 토큰**
**총합: ~530 토큰** (왕복 1회)

---

> **절약량: 71% 감소 (1,800 → 530 토큰)**
> 핵심: 모호한 요청은 되묻기를 유발 → 왕복 횟수 = 토큰 배수

## 예시 5. 출력 형식 미지정 vs 지정

### 낭비형: 형식 미지정

```
파이썬 리스트 메서드 알려줘
```

Claude 응답 (형식 없이 장문):
```
Python 리스트는 매우 유용한 자료구조입니다.
다양한 메서드를 제공하는데, 하나씩 살펴보겠습니다.

1. append() 메서드
append()는 리스트의 끝에 요소를 추가합니다.
예를 들어, my_list = [1, 2, 3]이 있을 때...
my_list.append(4)를 실행하면...

2. extend() 메서드
extend()는 다른 iterable의 모든 요소를...
```
**Output: ~800 토큰**

---

### 절약형: 형식 지정

```
파이썬 리스트 주요 메서드를 표로:
메서드명 | 기능 | 예시
(5개만)
```

Claude 응답:
```
| 메서드 | 기능 | 예시 |
|--------|------|------|
| append(x) | 끝에 추가 | [1,2].append(3) → [1,2,3] |
| pop(i) | i번째 제거 및 반환 | [1,2,3].pop(0) → 1 |
| sort() | 정렬 | [3,1,2].sort() → [1,2,3] |
| index(x) | x의 위치 반환 | [1,2,3].index(2) → 1 |
| count(x) | x 개수 반환 | [1,1,2].count(1) → 2 |
```
**Output: ~120 토큰**

---

> **절약량: 85% 감소 (800 → 120 토큰)**
> 핵심: 표, JSON, 코드만, N줄 이내 등 형식을 지정하면 output 토큰이 크게 줄어듦

## 절약 원칙 한눈에

```
1. 출력 형식 명시      → output 토큰 감소
2. 범위/조건 명시      → input 토큰 감소 + 재시도 감소
3. 새 대화 시작        → 히스토리 누적 방지
4. 파일 범위 지정      → 불필요한 input 제거
5. 한 번에 완전한 요청 → 왕복 횟수 감소
```
