## 작업 유형별 토큰 기준

> 이 기준을 초과하면 접근법을 재검토하세요.

| 작업유형 | 정상 범위 (총 토큰) | 경고 | 원인 분석 필요 |
|---------|-----------------|------|-------------|
| DOC | ~1,000 ~ 5,000 | 5,000 초과 | 문서가 너무 길거나 반복 작업 |
| CODE | ~500 ~ 3,000 | 3,000 초과 | 범위 재지정 필요 |
| DEBUG | ~300 ~ 2,000 | 2,000 초과 | 에러 메시지 없이 질문한 경우 |
| REVIEW | ~500 ~ 2,500 | 2,500 초과 | 파일 전체 읽힌 경우 |
| RESEARCH | ~1,000 ~ 4,000 | 4,000 초과 | Agent 위임 검토 |
| CHAT | ~100 ~ 1,000 | 1,000 초과 | 대화 히스토리 누적 확인 |

## 모델 선택 기준 (비용 최적화)

```
단순 질문/번역/요약         → Haiku 4.5   ($0.80/$4.00 per 1M)
일반 코딩/분석/문서 작업    → Sonnet 4.6  ($3.00/$15.00 per 1M)
복잡한 추론/아키텍처 설계   → Opus 4.6    ($15.00/$75.00 per 1M)
```

**비용 차이 예시 (input 2,000 + output 1,000 토큰 기준):**
- Haiku: ~$0.006
- Sonnet: ~$0.021  ← 현재 사용 중
- Opus: ~$0.105

→ 단순 작업을 Haiku로만 바꿔도 **세션당 71% 절약 가능**
