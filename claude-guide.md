## 1. 토큰이란?

Claude는 텍스트를 **토큰(token)** 단위로 처리합니다.
- 영어: 약 4글자 = 1토큰
- 한국어: 약 1~2글자 = 1토큰 (한국어는 토큰 소모가 더 많음)
- 이미지: 해상도에 따라 수백~수천 토큰

**토큰은 두 종류:**
| 종류 | 설명 |
|------|------|
| Input token | 내가 Claude에게 보내는 내용 (프롬프트 + 이전 대화 전체) |
| Output token | Claude가 응답하는 내용 |

## 2. 토큰 절약 핵심 원칙

### 2-1. 시스템 프롬프트를 짧고 정확하게
```
❌ 나쁜 예: "당신은 매우 친절하고 도움이 되는 AI 어시스턴트입니다. 항상 예의 바르게 대답하고..."
✅ 좋은 예: "Python 코드 리뷰 전문가. 버그와 성능 문제만 지적."
```

### 2-2. 대화 히스토리 관리
- Claude는 **대화 전체**를 매 턴마다 input으로 받음
- 긴 대화 = 토큰 폭증
- 새 주제는 **새 대화**로 시작하기
- Claude Code에서는 `/clear` 명령으로 컨텍스트 초기화

### 2-3. 불필요한 반복 제거
```
❌ "위에서 말한 코드를 다시 보여주고, 그 코드에서..."
✅ "위 코드에서 line 10만 수정해서..."
```

### 2-4. 파일 읽기 최소화
- 파일 전체를 읽히지 말고, 필요한 부분만 지정
- "파일 전체 읽어줘" → "파일의 50-100번째 줄만 읽어줘"

### 2-5. 구조화된 요청
- 원하는 것을 명확히 → 재시도 횟수 감소 → 토큰 절약
```
✅ "Python 함수 작성. 조건: 1) 입력: 정수 리스트 2) 출력: 중앙값 3) 빈 리스트 예외처리 포함"
```

## 3. Claude를 잘 사용하는 방법

### 3-1. 역할 부여 (Role Prompting)
Claude에게 역할을 주면 더 전문적인 답변이 나옵니다.
```
"당신은 10년 경력의 시니어 백엔드 개발자입니다. 다음 코드를 리뷰해주세요."
```

### 3-2. Chain of Thought (CoT)
복잡한 문제는 단계별로 생각하도록 유도:
```
"단계별로 생각하면서 답해주세요."
"먼저 문제를 분석하고, 그 다음 해결책을 제시해주세요."
```

### 3-3. Few-shot 예시 제공
원하는 형식이 있으면 예시를 보여주기:
```
입력 예시: "사과"
출력 예시: "fruit:사과:red"

이 형식으로 다음을 변환해주세요: "바나나"
```

### 3-4. 출력 형식 지정
```
"JSON 형식으로만 답해주세요. 설명 없이."
"마크다운 없이 순수 텍스트로."
"5줄 이내로 요약."
```

### 3-5. 반복 작업은 템플릿화
자주 쓰는 프롬프트는 파일로 저장해두고 재사용

## 4. Claude Code에서의 토큰 관리

### 4-1. 유용한 명령어
| 명령어 | 효과 |
|--------|------|
| `/clear` | 대화 히스토리 초기화 (토큰 리셋) |
| `/compact` | 대화를 요약으로 압축 |
| `/cost` | 현재 세션 토큰 사용량 확인 |
| `ESC ESC` | 실행 중인 작업 중단 |

### 4-2. CLAUDE.md 활용
프로젝트 루트에 `CLAUDE.md` 파일을 만들면 매 대화마다 자동으로 컨텍스트를 제공합니다.
→ 매번 설명하는 반복 토큰을 줄일 수 있음.

### 4-3. 모델 선택 전략
| 모델 | 용도 |
|------|------|
| Haiku 4.5 | 단순 작업, 빠른 응답, 저비용 |
| Sonnet 4.6 | 일반 코딩, 분석 (기본값, 균형 좋음) |
| Opus 4.6 | 복잡한 추론, 고품질 필요 시 |

## 5. Agent 안에서 Agent 사용하기 (Subagent)

### 5-1. 개념
Claude Code에서는 **Agent 도구**를 통해 하위 에이전트(subagent)를 생성할 수 있습니다.
메인 Claude가 복잡한 작업을 전문화된 하위 에이전트에게 위임하는 구조입니다.

```
메인 Claude
  ├── Explore Agent   → 코드베이스 탐색 전문
  ├── Plan Agent      → 구현 계획 설계 전문
  └── General Agent   → 복잡한 멀티스텝 작업
```

### 5-2. 사용 가능한 Agent 타입
| Agent 타입 | 특징 | 사용 시점 |
|------------|------|-----------|
| `general-purpose` | 모든 도구 사용 가능 | 복잡한 멀티스텝 작업 |
| `Explore` | 코드 탐색 특화 (수정 불가) | 파일/코드 검색, 구조 파악 |
| `Plan` | 구현 계획 설계 특화 | 아키텍처 설계, 단계별 계획 |
| `claude-code-guide` | Claude Code 기능 질문 | Claude 사용법, API 질문 |

### 5-3. Subagent의 토큰 절약 효과
- 메인 컨텍스트를 오염시키지 않음
- 대규모 검색 결과가 메인 대화에 쌓이지 않음
- 병렬 실행으로 작업 속도 향상

### 5-4. Subagent 사용 패턴

**병렬 실행 (독립 작업):**
```
"프론트엔드 코드 분석"과 "백엔드 코드 분석"을 동시에 두 Agent에게 위임
→ 시간 절약 + 컨텍스트 분리
```

**순차 실행 (의존 작업):**
```
1. Explore Agent → 코드베이스 파악
2. 그 결과를 바탕으로 Plan Agent → 구현 계획 수립
3. 그 계획을 바탕으로 메인 Claude → 코드 작성
```

**Background 실행:**
```
오래 걸리는 작업을 백그라운드로 보내고
메인은 다른 작업 진행 → 완료 시 알림
```

### 5-5. Worktree Isolation
민감하거나 위험한 작업은 `isolation: "worktree"` 옵션으로
임시 git worktree에서 격리 실행 가능.
→ 실험적 변경이 메인 브랜치에 영향 없음

## 6. API로 Agent 구현하기 (코드 예시)

```python
import anthropic

client = anthropic.Anthropic()

# 기본 호출
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "안녕하세요"}]
)

# 토큰 사용량 확인
print(f"Input tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
print(f"총 토큰: {response.usage.input_tokens + response.usage.output_tokens}")
```

```python
# 스트리밍으로 응답 (체감 속도 향상)
with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "긴 글 작성해줘"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

## 7. 체크리스트: 토큰 낭비 점검

- [ ] 원하는 출력 형식과 길이를 명시했나?
- [ ] 복잡한 탐색 작업은 Explore Agent에게 위임했나?
